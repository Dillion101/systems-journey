*Date: April 22, 2026*

## What a Character Array Is

A character array is how C represents strings. There is no built-in string type — a string is just an array of `char` values with a special terminator at the end.

```c
char line[1000];
```

This declares space for 999 characters plus one mandatory `\0` at the end.

---

## The Null Terminator — `\0`

Every C string ends with `\0` (the null character, ASCII value 0). This is how C knows where the string ends — not by tracking length separately, but by scanning until it hits `\0`.

```c
h e l l o \n \0
0 1 2 3 4  5  6
```

- `printf("%s", s)` prints characters from `s[0]` until it hits `\0`
- `\0` is never printed — it is purely a sentinel
- If you forget `\0`, `printf` keeps reading past the end of your array into whatever memory follows — undefined behaviour

This is why `my_getline` always ends with:

```c
s[i] = '\0';
```

And why placing `\0` earlier effectively shortens the string — everything after it becomes invisible to string functions.

---

## The Longest-Line Program

The program reads lines from input, tracks the longest one seen, and prints it at the end. It uses three components:

### `my_getline(char s[], int lim)`
Reads one line from stdin into `s[]`, up to `lim-1` characters. Appends `\n` if present, always appends `\0`. Returns the length including `\n`.

```c
int my_getline(char s[], int lim) {
    int c, i;
    for (i = 0; i < lim-1 && (c = getchar()) != EOF && c != '\n'; ++i)
        s[i] = c;
    if (c == '\n') {
        s[i] = c;
        ++i;
    }
    s[i] = '\0';
    return i;
}
```

### `copy(char to[], char from[])`
Copies a string character by character until it hits `\0`. The assignment `to[i] = from[i]` returns the character assigned — when that character is `\0`, the condition is false and the loop exits.

```c
void copy(char to[], char from[]) {
    int i = 0;
    while ((to[i] = from[i]) != '\0')
        ++i;
}
```

### `main()`
Calls `my_getline` in a loop. If the new line is longer than the current best, updates `max` and calls `copy` to save it into `longest[]`. After EOF, prints the longest line.

## Why `getline` Had to Be Renamed

The standard library on modern Linux defines its own `getline` function in `stdio.h`. K&R wrote this before that function existed. Naming your own function `getline` causes a type conflict at compile time:

```
error: conflicting types for 'getline'
```

The fix is to rename to something that doesn't clash — `my_getline` is the convention used here.

## Key Pattern: Placing `\0` to Trim a String

You can shorten a string without moving any characters — just place `\0` earlier. Everything after the new `\0` is invisible to all string functions.

Used in exercise 1-18 to strip trailing whitespace:

```c
// walk backwards past \n, spaces, tabs
for (i = len-1; line[i] == '\n' || line[i] == ' ' || line[i] == '\t'; i--)
    ;
line[i+1] = '\0';  // terminate right after the last real character
```

---

## Key Pattern: Two-Pointer Swap for Reversal

To reverse a string in place, use two indices — one from the left, one from the right — swapping characters and moving toward the centre. Stop when they meet (`i < j`).

```c
void reverse(char s[], int len) {
    int swap, i, j;
    i = 0;
    j = len - 2;   // len-1 is \n, len-2 is last real character
    for (; i < j; i++, j--) {
        swap = s[i];
        s[i] = s[j];
        s[j] = swap;
    }
}
```

Use `i < j` not `i != j` — with even-length strings, `i` and `j` cross each other without ever being equal, causing an infinite loop.

## Exercises

### Exercise 1-16 — Handle arbitrarily long lines

**Problem:** The original program silently truncates lines longer than `MAXLINE`. It reports `len = 999` even if the actual line is 5000 characters.

**Fix:** After the buffer fills, keep reading and counting characters without storing them. Return the true length.

```c
int my_getline(char s[], int lim) {
    int c, i, len;
    for (i = 0; i < lim-1 && (c = getchar()) != EOF && c != '\n'; ++i)
        s[i] = c;

    len = i;
    while (c != '\n' && c != EOF) {
        c = getchar();
        if (c != '\n' && c != EOF)
            ++len;
    }

    if (c == '\n') {
        s[i] = c;
        ++i;
    }
    s[i] = '\0';
    return len;
}
```

Verified with `python3 -c "print('a' * 1500)" | ./a.out` — prints 999 `a`s (as much as fits) while tracking true length of 1500.

### Exercise 1-17 — Print lines longer than 80 characters

Simplified from the longest-line program — no `max`, no `longest[]`, no `copy`. Just read each line and print if longer than 80:

while ((len = my_getline(line, MAXLINE)) > 0)
    if (len > 80)

### Exercise 1-18 — Remove trailing blanks/tabs, delete blank lines

Three phases per line:

1. Walk backwards from `len-1` while character is `\n`, space, or tab
2. Place `\0` at `i+1` to terminate the string
3. If `line[0] == '\0'` the line is blank — skip it. Otherwise print.

```c
int i;
for (i = len-1; line[i] == '\n' || line[i] == ' ' || line[i] == '\t'; i--)
    ;
line[i+1] = '\0';
if (line[0] != '\0')
    printf("%s\n", line);
```

**Key insight:** Declare `i` outside the `for` loop so it is accessible after the loop exits for the `\0` placement.

### Exercise 1-19 — Reverse a string, reverse input line by line

```c
void reverse(char s[], int len) {
    int swap = 0;
    int i = 0;
    int j = len - 2;   // skip \n at len-1
    for (; i < j; i++, j--) {
        swap = s[i];
        s[i] = s[j];
        s[j] = swap;
    }
}
```

**Bugs encountered:**
- `j` was initialised by calling `my_getline` again — reads extra input instead of using existing length
- `i != j` as loop condition — dangerous for even-length strings where pointers cross without meeting. Fixed to `i < j`
- `j = len-1` pointed at `\n` — newline got included in the reversal, moving to the front. Fixed to `j = len-2`
