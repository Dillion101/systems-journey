## What an Array Is in C

An array is a fixed-size, contiguous block of memory holding elements of the same type. You declare the size upfront and it never changes. There is no dynamic resizing, no bounds checking, and no safety net.

```c
int ndigit[10];  // 10 integers, indices 0 through 9
```

Array subscripts always start at zero. `ndigit[0]` is the first element, `ndigit[9]` is the last. Accessing `ndigit[10]` is undefined behaviour — C will not stop you, it will just read or write memory it should not.

---

## Always Initialise Arrays Explicitly

Local variables in C — including arrays — contain garbage values by default. Whatever was in that memory previously is what you get. Always zero-initialise before use:

```c
for (i = 0; i < 10; ++i)
    ndigit[i] = 0;
```

Confirmed in practice: skipping initialisation produced values like `-394056976` and `31858` in output.

---

## The ASCII Arithmetic Trick — `c - '0'`

Characters in C are integers. Digit characters have consecutive, ordered ASCII values:

| Character | ASCII value |
|-----------|-------------|
| `'0'` | 48 |
| `'1'` | 49 |
| `'2'` | 50 |
| ... | ... |
| `'9'` | 57 |

When `getchar()` returns `'6'`, `c` holds `54` — not `6`. The expression `c - '0'` converts the character into its integer value: `54 - 48 = 6`. That integer is used directly as an array index.

```c
++ndigit[c-'0'];
```

This is guaranteed to work by the C standard — digit characters are always consecutive in all character sets.

---

## The Index IS the Meaning

`ndigit[5]` is not just "the 6th slot" — it is specifically "the counter for the digit 5". The index and the domain value are the same thing. This is a recurring pattern in systems programming: use the value itself as the index to avoid any lookup or mapping step.

Exercise 1-14 takes this further — `freq[c]` uses the character's ASCII value directly as the index, no arithmetic needed.

---

## Buffer Overflow

If an index goes out of bounds, C does not panic or throw an exception. It writes into adjacent memory — corrupting whatever lives there. This is undefined behaviour and the root cause of a large class of real security vulnerabilities.

The fix is a bounds check before using any variable as an index:

```c
if (lenw < SIZE)
    ++word[lenw];
```

Words longer than `SIZE` are silently ignored — a design tradeoff. The alternative (dynamic allocation) is covered in Phase 1 with `malloc` and `realloc`.

---

## When Fixed Size Is Not Enough

If the input domain is unknown or unbounded — URLs, gene sequences, arbitrary user text — a fixed array size is the wrong tool. Dynamic allocation (`malloc`, `realloc`) is the C answer. This is why Phase 1 covers it immediately after arrays.

## Exercises

### Exercise 1-13 — Histogram of word lengths

**Task:** Print a horizontal histogram of the lengths of words in input.

**Key concepts:**
- IN/OUT state machine (same pattern as 1.5.4 word counting)
- Array index represents a measured quantity (word length)
- Bounds checking to prevent buffer overflow
- `%.*s` trick for printing N copies of a character

**Bugs encountered and fixed:**
- `lenw` uninitialised — garbage values caused wrong array indices
- State condition inverted — was checking `state == OUT` when `state == IN` was the correct trigger for word-end
- `lenw` never reset between words — carried previous word's length into the next
- `++word[i]` in print loop was incrementing counts while printing

**Final solution:**

```c
#include <stdio.h>

#define IN   1   /* inside a word */
#define OUT  0   /* outside a word */
#define SIZE 10

int main() {
    int c, lenw, state;
    int word[SIZE];
    state = OUT;
    lenw = 0;

    for (int i = 0; i < SIZE; i++)
        word[i] = 0;

    while ((c = getchar()) != EOF)
        if (c == ' ' || c == '\n' || c == '\t' || (c >= '0' && c <= '9')) {
            if (state == IN) {
                if (lenw < SIZE)    /* bounds check — buffer overflow prevention */
                    ++word[lenw];
                lenw = 0;
            }
            state = OUT;
        } else {
            lenw += 1;
            state = IN;
        }

    printf("Length of Word\n");
    for (int i = 0; i < SIZE; i++)
        printf("%d %.*s\n", i, word[i], "****************************************");
    printf("\n");
}
```

### Exercise 1-14 — Histogram of character frequencies

**Task:** Print a histogram of the frequencies of different characters in input.

**Key design decisions:**
- Array size is 256 — one slot per possible ASCII character value
- Index is `c` itself — the character's ASCII value IS the index, no arithmetic needed
- Only print rows where `freq[i] != 0` — avoids printing 200+ empty rows
- Use `%c` not `%d` to print the character label on each row

**Connection to arrays.c:** The digit counter used `c - '0'` to map a digit to index 0–9. This program removes that indirection entirely — `freq[c]++` works directly.

**Final solution:**

```c
#include <stdio.h>

#define SIZE 256

int main() {
    int c, freq[SIZE];

    for (int i = 0; i < SIZE; i++)
        freq[i] = 0;

    while ((c = getchar()) != EOF)
        ++freq[c];

    printf("Character Frequency Histogram\n");
    for (int i = 0; i < SIZE; i++)
        if (freq[i] != 0)
            printf("%c %.*s\n", i, freq[i], "***********************");
    printf("\n");
}
```
