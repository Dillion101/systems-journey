# K&R Section 1.8 — Arguments: Call by Value
*Date: April 22, 2026*

---

## What Call by Value Means

In C, when you call a function with arguments, the function receives **copies** of the values — not the originals. Modifying a parameter inside a function has no effect on the variable in the caller.

```c
void double_it(int x) {
    x = x * 2;   // modifies the local copy only
}

int main() {
    int n = 5;
    double_it(n);
    printf("%d\n", n);  // still prints 5
}
```

`x` inside `double_it` is a separate variable that starts with the value `5`. Changing it does not touch `n` in `main`.

This is different from Python, where mutable objects (lists, dicts) are passed by reference — modifying them inside a function affects the original. In C, **all scalar values are copied**.

---

## Why This Is Actually Useful

Because parameters are local copies, you can use them freely inside a function without worrying about corrupting the caller's data. K&R demonstrate this with `power`:

```c
int power(int base, int n)
{
    int p = 1;
    for (int i = 1; i <= n; ++i)
        p = p * base;
    return p;
}
```

`n` is used directly as the loop counter — it gets decremented each iteration. This is safe because `n` is a copy. The caller's variable is untouched.

An equivalent version that modifies `n` directly:

```c
int power(int base, int n)
{
    int p = 1;
    for (; n > 0; --n)
        p = p * base;
    return p;
}
```

Both versions produce the same result. The second is slightly more concise because it eliminates the separate loop counter `i`.

---

## The Exception: Arrays

Arrays are **not** copied when passed to functions. When you pass an array, the function receives a pointer to the first element — it is working on the original data directly.

```c
void fill(char s[], int len) {
    s[0] = 'X';   // modifies the original array in the caller
}
```

This is why `my_getline` and `copy` can fill and modify arrays passed to them — they are operating on the actual memory, not a copy.

This distinction — scalars are copied, arrays are not — is one of the most important things to internalise early in C. It becomes clearer when you study pointers in Phase 1.

---

## Summary

| What is passed | Behaviour |
|---|---|
| `int`, `char`, `double`, etc. | Copied — caller's variable unaffected |
| Arrays | Pointer to original — caller's data can be modified |
